# #child-modal 样式文件使用说明

## 功能概述
该样式文件定义了一个名为 `#child-modal` 的模态框组件样式，包含主模态框、确认弹窗及相关交互元素的视觉呈现，采用深色主题设计，带有蓝色系强调色，适合用于展示平台列表及操作确认场景。

## 结构说明
### 1. 主要组件
- **.modal-mask**：模态框遮罩层，全屏覆盖，半透明黑色背景
- **.modal**：主模态框容器，居中显示，包含头部、内容区
- **.confirm-modal**：确认弹窗容器，尺寸较小，用于操作确认

### 2. 子元素结构
- **.modal-header**：模态框头部
  - .modal-title：标题文本
  - .modal-close：关闭按钮
- **.modal-body**：内容区域，包含滚动条样式
  - .all-platforms-list：平台列表容器
  - .all-platform-item：单个平台项
    - .item-icon：平台图标容器
    - .item-info：平台信息区域
    - .item-meta：平台元数据（状态等）
    - .item-enter-btn：进入按钮
- **.modal-footer**：确认弹窗底部（仅确认弹窗使用）
  - .cancel-btn：取消按钮
  - .confirm-btn：确认按钮

## 样式特点
1. **主题风格**：深色背景（#0f172a）搭配蓝色系（#6495ed 系列）强调色，科技感风格
2. **交互反馈**：所有可交互元素（按钮、列表项）都有 hover 状态变化
3. **响应式设计**：模态框宽度使用百分比，最大宽度限制，适应不同屏幕
4. **细节处理**：
   - 自定义滚动条样式，与整体风格统一
   - 状态标签区分（active 绿色、maintenance 橙色）
   - 圆角设计和适当阴影，增强层次感

## 使用方法
1. 将该样式应用于 ID 为 "child-modal" 的容器元素
2. 根据需要展示的内容，选择使用：
   - 主模态框：使用 .modal 类容器，包含 .modal-header 和 .modal-body
   - 确认弹窗：使用 .confirm-modal 类容器，额外包含 .modal-footer
3. 平台列表项需按照 .all-platform-item 的结构组织内容
4. 状态标签通过添加 .active 或 .maintenance 类来展示不同状态

## 注意事项
1. 模态框使用 fixed 定位，会脱离文档流，需注意层级（z-index: 1000）
2. 主模态框高度固定为视口的 80%，内容超出时会显示滚动条
3. 确认弹窗高度自适应内容，无滚动条
4. 关闭按钮和遮罩层可绑定关闭事件，实现模态框隐藏功能
